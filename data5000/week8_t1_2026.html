Part 1

What is AI-Assisted Programming?

Understanding the new paradigm for business analysts

AI-Assisted Programming: Definition

AI-Assisted Programming (also known as "Vibe Coding") is the practice of using Large Language Models to generate, debug, and refine code based on natural-language descriptions of what you want to accomplish.

Traditional Programming

The analyst must learn syntax, libraries, and language-specific conventions. Errors require manual debugging. The bottleneck is technical knowledge.

AI-Assisted Programming

The analyst describes the problem in plain language. AI generates the code. The analyst validates the output. The bottleneck is problem understanding.

The fundamental shift: from "How do I write this code?" to "What do I need this code to do?"

The Human–AI Collaboration Spectrum

AI-Assisted Programming sits in the middle — where the analyst maintains control of the thinking while AI handles syntax.

As a business analyst, you operate in the centre of this spectrum. You bring the domain expertise and business judgement; AI brings the technical implementation. Neither succeeds alone.

Why Business Analysts Need AI-Assisted Programming

92% of developers now use AI coding tools (GitHub, 2024)
55% faster task completion with AI assistance (McKinsey, 2024)
70% of new code at Google is AI-assisted (Sundar Pichai, 2025)

For Business Analysts

You no longer need a Computer Science degree to perform sophisticated data analysis. AI-assisted programming democratises access to analytical tools — your competitive advantage is now your business understanding, not your coding ability.

The Catch

AI-generated code can be wrong, misleading, or inefficient. The analyst must evaluate outputs critically. Blindly trusting AI code is more dangerous than writing no code at all — it creates false confidence in flawed analysis.

Part 2

The Vibe Coding Workflow

A structured methodology for AI-assisted business analysis

The 5-Step Vibe Coding Workflow

A repeatable framework for solving business problems with AI-generated code.

1 Frame Human
→
2 Describe Human
→
3 Generate AI
→
4 Validate Both
→
5 Interpret Human

Notice: Steps 1, 2, and 5 are entirely human. Only Step 3 is AI-driven. This is where the business analyst adds irreplaceable value — AI cannot frame a business question or interpret results for stakeholders.

Step 1: Frame the Business Question

Before writing any prompt, articulate precisely what you need to know. A vague question produces vague code.

Vague Framing

"Analyse the customer data and find something interesting."

Precise Framing

"Which customer segments have the highest delivery cost relative to their order value, and what patterns distinguish profitable from unprofitable customers?"

Framing Checklist

✓

Specific metric: What exactly are you measuring?

✓

Comparison: Between which groups or time periods?

✓

Decision relevance: How will the answer inform a business action?

✓

Scope: What data is available to answer this?

Step 2: Describe Your Data to the AI

The AI has never seen your dataset. You must provide a precise description for it to generate correct code.

Insufficient Description
"I have customer data in a CSV. Analyse it."

AI does not know the column names, data types, or structure. It will guess — and likely guess wrong.

Effective Description
"I have a CSV file with 5,000 rows of grocery delivery data. Columns include: customer_id, order_date, delivery_zone (A/B/C/D), order_total (float), delivery_cost (float), items_count (int), customer_type (new/returning), and delivery_time_mins (int)."

Pro Tip: Share the First Few Rows

Use df.head() and paste the output into your prompt. This gives the AI concrete examples of your data format, removing all ambiguity about column names and value types.

Step 3: Generate Code Iteratively

Do not ask for an entire analysis at once. Build incrementally — each prompt adds one layer of complexity.

Round 1: Explore

"Load the CSV and show me summary statistics for order_total and delivery_cost. Include the distribution shape."

Start simple. Verify data loads correctly.

Round 2: Analyse

"Group by delivery_zone and calculate average order_total, delivery_cost, and profit margin. Show as a bar chart."

Add analytical depth. Check results make sense.

Round 3: Deepen

"For the unprofitable zone, segment by customer_type and items_count. What patterns drive the losses?"

Targeted investigation. Build toward recommendations.

Each round builds on verified results from the previous round. This is the core rhythm of vibe coding: prompt → run → verify → prompt again.

Step 4: Validate Against Business Logic

Code that runs without errors is not the same as code that produces correct results. Use these validation checks:

Sanity Checks

✓

Range check: Are values within realistic bounds? (e.g., delivery cost should not be negative)

✓

Row count: Does the output have the expected number of rows? Did a merge drop data?

✓

Totals check: Do group subtotals sum to the overall total?

✓

Directional logic: If we filtered high-value customers, is their average order higher?

Ask AI to Validate Itself

"The code shows that Zone D has an average delivery cost of $2.50. This seems too low for a remote zone. Can you verify the calculation and check if there are data quality issues?"

Use your business intuition to challenge suspicious results. AI will often find its own errors when you point out that something looks wrong.

Step 5: Interpret for Business Decisions

Code and charts are not the deliverable. Business recommendations are.

From Technical Output...

Zone Avg_Cost Avg_Revenue Margin A $4.20 $45.80 91% B $6.10 $38.20 84% C $9.80 $32.50 70% D $14.30 $28.10 49%

...To Business Recommendation

Finding: Zone D delivery costs consume 51% of revenue — 3.4× higher than Zone A.

Recommendation: Introduce a minimum order value of $40 for Zone D, or consolidate deliveries to reduce per-order costs. Projected margin improvement: 49% → 65%.

This is the step AI cannot do for you. Translating numbers into strategic action requires understanding of the business context, stakeholder priorities, and operational constraints that only a human analyst possesses.

Good vs. Poor Prompts for Code Generation

The quality of your prompt directly determines the quality of the generated code.

Poor Prompt
"Make a chart of the data"

Which data? What type of chart? What's on each axis? The AI must guess everything.

Poor Prompt
"Do a complete analysis of customer behaviour including segmentation, prediction, clustering, and a dashboard"

Too much at once. The AI will produce long, untestable code with compounding errors.

Effective Prompt
"Create a horizontal bar chart showing average delivery_cost by delivery_zone, sorted highest to lowest. Use a red colour palette. Label each bar with the dollar value."

Specific chart type, exact column names, sort order, colour, and labels — no guessing required.

Effective Prompt
"Group the dataframe by customer_type and calculate the mean and median of order_total. Show results as a table."

One clear task. Easy to verify. Builds incrementally toward deeper analysis.

The Art of Iterative Prompting

Effective vibe coding follows a conversation pattern — each prompt builds on the last, like a dialogue with a colleague.

Each round follows: Prompt → Run → Verify → Next Prompt. Never skip verification.

Prompt Patterns for Business Data Analysis

Reusable prompt templates for common analytical tasks. Adapt these to your own datasets.

Exploration

"Show the shape, column names, data types, and first 5 rows of the dataframe. Flag any columns with missing values."
"Create a correlation matrix heatmap for all numerical columns. Annotate with values."

Analysis

"Group by [column] and calculate mean, median, and count of [metric]. Sort by mean descending."
"Compare [metric] between [group A] and [group B] using a box plot. Add a t-test for significance."

Visualisation

"Create a scatter plot of [X] vs [Y], coloured by [category]. Add a trend line and R² value."
"Show the distribution of [column] as a histogram with 20 bins. Overlay a KDE curve."

Key principle: Always use your actual column names in prompts. Generic placeholders like "the data" or "the variable" force the AI to guess and increase error rates.

Part 3

Trust But Verify

Critical evaluation of AI-generated code

Why Critical Evaluation Matters

AI-generated code has a unique danger: it often looks correct and runs without errors, yet produces wrong results.

Runs ✓ Code executes without any error messages
Looks ✓ Output appears professionally formatted
Wrong ✗ The underlying calculation or logic is incorrect

Real-World Consequence

An analyst presents a chart showing Customer Segment A is most profitable. The CEO shifts budget toward Segment A. But the AI code calculated revenue instead of profit — Segment A actually has the lowest margins. The decision costs the company $2M. The code ran perfectly. The business logic was wrong.

Common AI Code Errors to Watch For

Wrong Column Used

AI uses revenue when you meant profit, or order_date when you meant delivery_date.

Silent Data Loss

A merge or filter silently drops rows. Your analysis covers 3,000 of 5,000 customers without warning.

Hallucinated Functions

AI invents a function that does not exist in the library, or uses deprecated syntax that was removed in recent versions.

Incorrect Aggregation

Uses mean when the business question requires sum, or calculates a simple average when a weighted average is needed.

Misleading Visualisation

Y-axis does not start at zero, truncating differences. Or chart type obscures the actual pattern in the data.

Correlation ≠ Causation

AI code may present correlations as causal relationships — a pattern you learned to recognise in Week 4 (Causal AI).

Spot the Error: Example 1

The business question: "What is the average profit margin by delivery zone?"

AI-Generated Code
# Calculate average profit margin by zone margins = df.groupby('delivery_zone')['order_total'].mean() margins.plot(kind='bar', title='Avg Profit Margin by Zone')

This code runs without error and produces a clean bar chart. But can you spot the problem?

The Error

The code calculates average order_total (revenue), not profit margin. Profit margin requires: (order_total - delivery_cost) / order_total. The chart title says "Profit Margin" but shows revenue — a dangerously misleading output.

How to Catch It

Ask yourself: "Is this calculating what I actually asked for?" Check if the column used matches the business concept. Profit margin is a ratio — any output showing dollar amounts is likely wrong.

Spot the Error: Example 2

The business question: "How many customers are in each loyalty tier?"

AI-Generated Code
# Count customers per loyalty tier tier_counts = df.groupby('loyalty_tier')['customer_id'].count() print(tier_counts) # Output: # Gold 2,847 # Silver 4,231 # Bronze 8,156 # Total: 15,234
The Error

The dataset has 5,000 unique customers, but the code counted 15,234 orders. Because each customer has multiple orders, .count() on the orders table counts rows, not unique customers. The correct approach is .nunique().

How to Catch It

Row count check: "I know I have 5,000 customers. If the total exceeds 5,000, something is counting orders, not customers." Always compare totals against known figures.

Collaborative Debugging with AI

When code produces unexpected results, use AI to help debug — but guide it with your business knowledge.

Effective Debugging Prompts

"The output shows 15,234 total customers but I know the dataset has only 5,000 unique customers. The code uses .count() on customer_id after groupby. What is wrong and how do I fix it?"
"This chart shows Zone D with the lowest delivery cost, but Zone D is our most remote zone. Can you add a check for how many rows are in each zone — is it possible we have very little data for Zone D?"

The Pattern

1

State what you expected based on business knowledge

2

State what you got from the code output

3

Hypothesise why there is a mismatch

4

Ask AI to investigate your hypothesis

This mirrors how experienced analysts debug — you never just say "it's broken," you say "I expected X but got Y, which suggests Z."

Workshop: FreshCart Analytics Challenge

Hands-on AI-assisted business analysis using Google Colab + Gemini

Business Scenario: FreshCart Grocery Delivery

The Company

FreshCart is a mid-sized grocery delivery startup operating across four delivery zones (A, B, C, D). They have 5,000 customers and have been operating for two years.

The Problem

Despite growing order volume, FreshCart is losing money on certain delivery routes and customer segments. The CEO needs to know: where are we losing money, why, and what should we do about it?

Your Role

You are a business analytics consultant. Using AI-assisted programming in Google Colab, analyse FreshCart's data and produce actionable recommendations for the executive team.

Your Tools

Google Colab with Gemini AI integration. You will use the 5-step vibe coding workflow to explore data, generate insights, and build your recommendations.

FreshCart Dataset Structure

Your dataset contains 5,000 customer records with the following columns:

Column Type Description Example Values
customer_idStringUnique customer identifierFC-001, FC-002
delivery_zoneCategoryGeographic delivery areaA, B, C, D
customer_typeCategoryCustomer classificationNew, Returning
order_totalFloatTotal order value in dollars$12.50 – $185.00
delivery_costFloatCost to deliver the order$3.00 – $18.00
items_countIntegerNumber of items in order2 – 45
delivery_time_minsIntegerTime from order to delivery15 – 120
satisfaction_scoreFloatPost-delivery rating (1–5)1.0 – 5.0

Workshop: Three-Phase Challenge

Phase 1: Explore (15 min)

Goal: Understand the data landscape

Use AI to load data, generate summary statistics, and create initial visualisations. Answer: Who are FreshCart's customers? What does the order distribution look like? Which zones have the most activity?

Deliverable: 2–3 code cells with exploration outputs

Phase 2: Diagnose (25 min)

Goal: Identify where and why money is lost

Calculate profit margins by zone and customer type. Find which segments are unprofitable. Investigate what drives high delivery costs. Create comparison visualisations.

Deliverable: 3–4 code cells with diagnostic analysis

Phase 3: Recommend (20 min)

Goal: Propose data-driven business actions

Translate your findings into specific recommendations. What should FreshCart change? What minimum order value would fix Zone D? Which customer segments should receive retention offers?

Deliverable: Executive summary with 3 recommendations

Submit both: (1) your working Colab notebook and (2) a short business report with your top 3 data-backed recommendations.
