Week 1 Recap: AI vs Machine Learning

Artificial Intelligence

Learning Paradigms: Supervised vs Unsupervised

Supervised Learning

  • Training Data: Has correct answers (labels)
  • Goal: Learn mapping from input to output
  • Types: Classification & Regression
  • Examples: Email spam detection, house price prediction
  • Evaluation: Compare predictions to actual labels

Unsupervised Learning

  • Training Data: No correct answers provided
  • Goal: Discover hidden patterns in data
  • Types: Clustering & Association
  • Examples: Customer segmentation, market basket analysis
  • Evaluation: Domain expertise and business value

Understanding Data Types in Machine Learning

Numerical Data

Continuous: Can take any value within a range

Examples: Age (25.5 years), Income ($45,230.50), Temperature (23.7°C)

Discrete: Whole numbers only

Examples: Number of children (2), Items purchased (5), Years of experience (3)

Categorical Data

Nominal: Categories with no natural order

Examples: Gender (Male/Female), Color (Red/Blue/Green), Department (Sales/Marketing/IT)

Ordinal: Categories with meaningful order

Examples: Education (High School < Bachelor < Master), Rating (Poor < Good < Excellent)

Binary Data

Special case of categorical with only two values

Examples: Purchased (Yes/No), Email opened (True/False), Gender (Male/Female)

Why important: Often used as target variables in classification

Text Data

Unstructured text that needs special processing

Examples: Customer reviews, Email content, Social media posts

Challenge: Must convert to numerical format for machine learning

What is Supervised Learning?

Definition

Supervised learning is a family of algorithms trained on feature-label pairs to predict unseen labels. It's like learning with a teacher who shows you examples and correct answers.

Key Components

  • Features (X): Input variables used to make predictions
  • Labels/Targets (Y): Output variables we want to predict
  • Training Data: Historical examples with known outcomes
  • Model: The algorithm that learns patterns

Two Main Types

  • Classification: Predicting categories (Will customer buy? Yes/No)
  • Regression: Predicting numerical values (How much will customer spend?)

When to Use Each Algorithm

  • Linear/Logistic Regression: Simple relationships, need interpretability
  • Decision Tree: Need clear rules, easy to explain
  • Random Forest: Want high accuracy, don't mind complexity
  • KNN: Simple patterns, small to medium datasets
  • SVM: Complex patterns, medium datasets

Key Considerations

  • Accuracy: How precise are the predictions?
  • Speed: How fast does it train and predict?
  • Interpretability: Can you explain why it made a decision?
  • Data Requirements: How much data does it need?

Evaluating Classification Models

How do we know which algorithm performs better?

We use specific metrics to measure and compare classification model performance

Accuracy

What it measures: Percentage of correct predictions

Formula: Correct Predictions / Total Predictions
Example: 850 correct out of 1000 = 85% accuracy

Best for: Balanced datasets with equal class sizes

Precision

What it measures: Of predicted positives, how many were actually positive?

Example: Of 100 emails predicted as spam, 90 were actually spam = 90% precision

Business impact: Reduces false alarms

Recall (Sensitivity)

What it measures: Of actual positives, how many were correctly identified?

Example: Of 200 actual spam emails, 180 were caught = 90% recall

Business impact: Reduces missed opportunities/threats

F1-Score

What it measures: Balance between Precision and Recall

Harmonic mean of Precision and Recall
Good when you need both metrics to be high

Best for: Unbalanced datasets

Evaluating Regression Models

For numerical predictions, we measure how close our predictions are to actual values

Mean Absolute Error (MAE)

  • What it measures: Average absolute difference between predictions and actual values
  • Example: Predicted house price $450K, actual $480K → Error = $30K
  • Advantage: Easy to interpret in original units
  • Business use: "On average, our predictions are off by $30K"

Root Mean Square Error (RMSE)

  • What it measures: Penalizes large errors more heavily
  • Example: $10K error vs $50K error - RMSE gives much higher penalty to $50K
  • Advantage: Sensitive to outliers
  • Business use: When large errors are particularly costly

R-squared (R²)

Measures: How much of the variance in target variable is explained by features

R² = 0.85 means 85% of house price variation is explained by size, location, age, etc.
Higher R² = Better model (closer to 1.0 is better)

Choosing Algorithms Based on Business Impact

1. Define Business Cost

What does a wrong prediction cost your business?

2. Choose Right Metric

Select evaluation metric that aligns with business goals

3. Compare Algorithms

Test multiple algorithms using your chosen metric

Example: Email Spam Detection

  • False Positive: Good email marked as spam (customer angry)
  • False Negative: Spam email gets through (minor annoyance)
  • Business Priority: High precision to avoid blocking good emails
  • Algorithm Choice: Choose model with 95%+ precision, even if recall is lower

Example: Medical Diagnosis

  • False Positive: Healthy person diagnosed with disease (unnecessary treatment)
  • False Negative: Sick person not diagnosed (potentially fatal)
  • Business Priority: High recall to catch all cases
  • Algorithm Choice: Choose model with 95%+ recall, even if precision is lower

Framework for Algorithm Selection

Step-by-Step Decision Process

Business Context

Cost of errors, interpretability needs

Data Analysis

Size, quality, feature types

Performance Testing

Cross-validation with multiple algorithms

Final Selection

Best performance + business requirements

High Interpretability Needed

  • Best Choice: Decision Trees
  • Why: Clear rules, easy to explain
  • Use Case: Medical diagnosis, loan approval

Maximum Accuracy Needed

  • Best Choice: Random Forest or SVM
  • Why: Handle complex patterns
  • Use Case: Image recognition, fraud detection

Simple Relationships

  • Best Choice: Linear/Logistic Regression
  • Why: Fast, interpretable, robust
  • Use Case: Price prediction, basic classification

Small Dataset

  • Best Choice: K-Nearest Neighbors
  • Why: No training required, simple
  • Use Case: Recommendation systems, local patterns
