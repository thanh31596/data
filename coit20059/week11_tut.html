Objectives & Prerequisites

What you will build

A complete Jakarta EE RESTful web service that performs CRUD operations on a Product entity, backed by a MySQL database and deployed on GlassFish 7.

By the end of this lab you will be able to:

  • Create a JAX-RS resource class with @Path, @GET, @POST, @PUT, @DELETE
  • Use @PathParam and @QueryParam to accept input
  • Return proper HTTP status codes using the Response class
  • Test your API endpoints using a browser and curl
Pre-lab checklist — confirm before starting:
  • MySQL is running (check via mysqladmin ping or MySQL Workbench)
  • GlassFish 7 is running (http://localhost:4848 loads the Admin Console)
  • JDBC connection pool and resource are configured in GlassFish (from earlier labs)
  • Apache NetBeans is open and can see GlassFish under the Services → Servers tab

Open pom.xml from the Project Files node. Ensure these dependencies are present inside <dependencies>:

<dependency>
    <groupId>jakarta.platform</groupId>
    <artifactId>jakarta.jakartaee-api</artifactId>

</dependency>

<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
    <version>8.0.33</version>

Why provided scope? GlassFish already contains the Jakarta EE libraries. The provided scope means "I need this to compile, but the server will provide it at runtime." This keeps your WAR file small.

Step 3 — Create the Product Entity

3.1 — Create the class
Right-click the package au.edu.cqu.productrest → New → Java Class.
Name it Product. Click Finish.

Replace the file contents with:

package au.edu.cqu.productrest;

import jakarta.persistence.*;
import java.io.Serializable;

@Entity
@Table(name = "products")
public class Product implements Serializable {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String name;

    private String description;

    @Column(nullable = false)
    private double price;

    // Default constructor (required by JPA)
    public Product() {}

    // Getters and setters
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public String getDescription() { return description; }
    public void setDescription(String d) { this.description = d; }
    public double getPrice() { return price; }
    public void setPrice(double price) { this.price = price; }
}

Step 4 — Create persistence.xml

4.1 — Create the file
Right-click src/main/resources → New → Folder → name it META-INF.
Right-click META-INF → New → XML Document → name it persistence → Click Finish.

Replace the entire file with:

<?xml version="1.0" encoding="UTF-8"?>
<persistence version="3.0"
  xmlns="https://jakarta.ee/xml/ns/persistence"
  xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation="https://jakarta.ee/xml/ns/persistence
    https://jakarta.ee/xml/ns/persistence/persistence_3_0.xsd">

  <persistence-unit name="ProductPU"
                     transaction-type="JTA">
    <jta-data-source>jdbc/myDatasource</jta-data-source>
    <class>au.edu.cqu.productrest.Product</class>
    <properties>
      <property name="jakarta.persistence.schema-generation.database.action"
                value="create"/>
    </properties>
  </persistence-unit>
</persistence>
Critical: The <jta-data-source> value must match the JDBC Resource JNDI name you configured in GlassFish Admin Console (http://localhost:4848 → Resources → JDBC → JDBC Resources). If yours is different, update it here.
schema-generation...action = "create" tells JPA to auto-create the products table on first deployment. Change to "none" after the table exists to avoid data loss on redeployment.

Step 5 — Create the JAX-RS Application Config

JAX-RS needs a small configuration class that tells the server "this project has REST endpoints" and defines the base URL path.

5.1 — Create the class
Right-click the package au.edu.cqu.productrest → New → Java Class.
Name: RestApplication. Click Finish. 
package au.edu.cqu.productrest;

import jakarta.ws.rs.ApplicationPath;
import jakarta.ws.rs.core.Application;

@ApplicationPath("/api")       // All REST URLs start with /api
public class RestApplication extends Application {
    // No code needed — the annotation does all the work
}
What this does: The @ApplicationPath("/api") annotation registers the JAX-RS servlet with GlassFish and sets /api as the base path. So a resource at @Path("/products") becomes accessible at:

http://localhost:8080/ProductRestService/api/products

Step 6 — Create the ProductResource (GET endpoints)

6.1 — Create the class
Right-click the package → New → Java Class → Name: ProductResource.

Start with GET endpoints only — we'll add POST, PUT, DELETE next:

package au.edu.cqu.productrest;

import jakarta.persistence.*;
import jakarta.ws.rs.*;
import jakarta.ws.rs.core.*;
import java.util.List;

@Path("/products")
@Produces(MediaType.APPLICATION_JSON)
@Consumes(MediaType.APPLICATION_JSON)
public class ProductResource {

    @PersistenceContext(unitName = "ProductPU")
    private EntityManager em;

    // GET /api/products — return all products
    @GET
    public List<Product> getAll() {
        return em.createQuery("SELECT p FROM Product p",
                              Product.class).getResultList();
    }

    // GET /api/products/5 — return one product by ID
    @GET
    @Path("/{id}")
    public Response getById(@PathParam("id") Long id) {
        Product p = em.find(Product.class, id);
        if (p == null) {
            return Response.status(404).build();
        }
        return Response.ok(p).build();
    }
}

Step 8 — Add the POST Endpoint (Create)

Add this method inside ProductResource.java, below the GET methods:

// POST /api/products — create a new product
@POST
@Transactional
public Response create(Product product) {
    em.persist(product);
    return Response
        .status(Response.Status.CREATED)   // 201
        .entity(product)
        .build();
}
Add this import at the top of the file:
import jakarta.transaction.Transactional;

@Transactional tells the container to wrap this method in a database transaction. Without it, em.persist() will fail because there's no active transaction.

Step 9 — Add PUT and DELETE Endpoints

Add both methods inside ProductResource.java:

// PUT /api/products/5 — update an existing product
@PUT
@Path("/{id}")
@Transactional
public Response update(@PathParam("id") Long id,
                       Product updated) {
    Product existing = em.find(Product.class, id);
    if (existing == null) {
        return Response.status(404).build();
    }
    existing.setName(updated.getName());
    existing.setDescription(updated.getDescription());
    existing.setPrice(updated.getPrice());
    em.merge(existing);
    return Response.ok(existing).build();       // 200
}

// DELETE /api/products/5 — delete a product
@DELETE
@Path("/{id}")
@Transactional
public Response delete(@PathParam("id") Long id) {
    Product p = em.find(Product.class, id);
    if (p == null) {
        return Response.status(404).build();
    }
    em.remove(p);
    return Response.noContent().build();      // 204
}

Troubleshooting Guide

SymptomLikely CauseFix
404 on all URLs RestApplication.java missing or @ApplicationPath wrong Verify the class exists and extends Application. Check the context root in GlassFish Admin Console.
500 Internal Server Error Persistence unit not found or JDBC pool broken Check persistence.xml is in src/main/resources/META-INF/. Verify the JNDI name matches GlassFish. Ping the JDBC pool in Admin Console.
POST returns 415 Unsupported Media Type Missing Content-Type header in curl Add -H "Content-Type: application/json" to your curl command.
POST returns 500 — "no transaction" Missing @Transactional annotation Add @Transactional to POST, PUT, DELETE methods. Import jakarta.transaction.Transactional.
JSON response is empty {} Entity fields are missing getters JSON-B uses getter methods. Ensure every field has a public getXxx() method.
Table not created in MySQL schema-generation property missing or wrong Set database.action to "create" in persistence.xml. Check MySQL user has CREATE TABLE permission.

Knowledge Check

Q1: Your REST resource is at @Path("/products") and your config has @ApplicationPath("/api"). The app is deployed as "ProductRestService". What is the full URL?
The URL is: context root (/ProductRestService) + application path (/api) + resource path (/products).
Q2: What happens if you remove @Transactional from the POST method?
em.persist() writes to the database, which requires a JTA transaction. @Transactional tells the container to start one automatically.
Q3: After a successful DELETE, what HTTP status code should your endpoint return?
204 No Content is the standard response for a successful DELETE — the operation succeeded and there is no response body to return.

Extension Task 1 — Search by name

Add a @QueryParam-based search to the GET-all endpoint. If the caller provides ?name=Laptop, filter results to products whose name contains that string.

@GET
public List<Product> getAll(
    @QueryParam("name") String name) {
  if (name != null) {
    return em.createQuery(
      "SELECT p FROM Product p " +
      "WHERE p.name LIKE :n",
      Product.class)
      .setParameter("n", "%"+name+"%")
      .getResultList();
  }
  return em.createQuery(
    "SELECT p FROM Product p",
    Product.class).getResultList();
}
