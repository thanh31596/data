P1

ORM Customisation

Customise table and column mappings for a Book entity, then persist it to MySQL.

Project 1 — Requirements

You have a plain Book class. Your task is to make it a JPA entity and customise these specific mappings:

Customisation Annotation What It Does
Table name → MyFavouriteBook@Table(name=...)Overrides default BOOK table name
title column → Book_Title, NOT NULL, not updatable@Column(name=..., nullable=false, updatable=false)Rename, prevent nulls, lock after insert
description length → 2,000 chars@Column(length=2000)Override default VARCHAR(255)
nbOfPage column → Number_of_Pages, NOT NULL@Column(name=..., nullable=false)Rename and prevent nulls

Step 2 — Create Book.java

Create this file in your project's source package. The highlighted lines are the customisation annotations.

package week2.project1;

import jakarta.persistence.*;

@Entity
@Table(name = "MyFavouriteBook")
public class Book {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "Book_Title", nullable = false, updatable = false)
    private String title;

    private Float price;

    @Column(length = 2000)
    private String description;

    private String isbn;

    @Column(name = "Number_of_Pages", nullable = false)
    private Integer nbOfPage;

    private Boolean illustrations;

    // Required no-arg constructor
    public Book() {}

    // Convenience constructor
    public Book(String title, Float price, String description,
               String isbn, Integer nbOfPage, Boolean illustrations) {
        this.title = title;
        this.price = price;
        this.description = description;
        this.isbn = isbn;
        this.nbOfPage = nbOfPage;
        this.illustrations = illustrations;
    }

    // Getters and setters (generate via IDE: right-click → Insert Code)
    // ... getId(), getTitle(), setTitle(), getPrice(), setPrice(), etc.
}

Step 3 — Configure persistence.xml

Create src/main/resources/META-INF/persistence.xml. This tells JPA where the database is and which entities to manage.

<?xml version="1.0" encoding="UTF-8"?>
<persistence version="3.0"
  xmlns="https://jakarta.ee/xml/ns/persistence">

  <persistence-unit name="Week2PU"
                    transaction-type="RESOURCE_LOCAL">
    <provider>org.eclipse.persistence.jpa.PersistenceProvider</provider>
    <class>week2.project1.Book</class>
    <properties>
      <property name="jakarta.persistence.jdbc.driver"
               value="com.mysql.cj.jdbc.Driver"/>
      <property name="jakarta.persistence.jdbc.url"
               value="jdbc:mysql://localhost:3306/week2db"/>
      <property name="jakarta.persistence.jdbc.user"
               value="root"/>
      <property name="jakarta.persistence.jdbc.password"
               value="your_password"/>
      <property name="jakarta.persistence.schema-generation.database.action"
               value="drop-and-create"/>
    </properties>
  </persistence-unit>
</persistence>
Before running: Create the week2db database in MySQL first: CREATE DATABASE week2db;
Why drop-and-create? During development, this regenerates tables from your entity annotations every time you run. Switch to none for production.

Step 4 — Create Main.java

This class creates an EntityManager, begins a transaction, persists a Book, and commits.

package week2.project1;

import jakarta.persistence.*;

public class Main {
    public static void main(String[] args) {
        // 1. Create EntityManagerFactory from persistence.xml
        EntityManagerFactory emf =
            Persistence.createEntityManagerFactory("Week2PU");
        EntityManager em = emf.createEntityManager();

        // 2. Begin transaction
        EntityTransaction tx = em.getTransaction();
        tx.begin();

        // 3. Create and persist a Book
        Book book = new Book(
            "Beginning Jakarta EE",
            49.99f,
            "A comprehensive guide to enterprise Java",
            "978-1-4302-4627-5",
            560,
            true
        );
        em.persist(book);
        System.out.println("Book persisted with ID: " + book.getId());

        // 4. Commit and clean up
        tx.commit();
        em.close();
        emf.close();
    }
}

Troubleshooting — Project 1

Table not found / schema not created

Check that persistence.xml is in src/main/resources/META-INF/ (not just META-INF). Verify the database URL and that week2db exists.

Column "Book_Title" cannot be null

You set nullable = false — make sure you pass a non-null title in your Book constructor.

No Persistence provider for EntityManager

EclipseLink dependency may be missing from pom.xml. Also check the persistence unit name matches between persistence.xml and Main.java.

Table still named BOOK

Make sure @Table(name = "MyFavouriteBook") is on the class, not on a field. Also run a clean build to clear cached schema.

Create a one-to-many relationship between AnOrder and OrderLine, then persist and verify.

Project 2 — What We're Building

An order that contains multiple order lines — a classic one-to-many unidirectional relationship.

AnOrder (Java)

creationDate
orderLines → List<OrderLine>
→
OrderLine (Java)
id
item

Navigation: Order → OrderLines only (unidirectional)

What JPA will create: Three database tables — ANORDER, ORDERLINE, and a join table ANORDER_ORDERLINE linking them via foreign keys. You don't need to create these manually.

Step 1 — Create OrderLine.java

This is the simpler entity — it doesn't know about its parent Order (unidirectional).

package week2.project2;

import jakarta.persistence.*;

@Entity
public class OrderLine {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String item;
    private Double unitPrice;
    private Integer quantity;

    // Required no-arg constructor
    public OrderLine() {}

    // Convenience constructor
    public OrderLine(String item, Double unitPrice, Integer quantity) {
        this.item = item;
        this.unitPrice = unitPrice;
        this.quantity = quantity;
    }

    // Getters and setters...
    // Generate via IDE: right-click → Insert Code → Getter and Setter
}
No relationship annotations here. Since the relationship is unidirectional from Order → OrderLine, the OrderLine entity has no reference back to its parent order.

Step 2 — Create AnOrder.java

This entity "owns" the relationship. The List<OrderLine> field triggers JPA's one-to-many behaviour.

package week2.project2;

import jakarta.persistence.*;
import java.util.*;

@Entity
public class AnOrder {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Temporal(TemporalType.TIMESTAMP)
    private Date creationDate;

    // Collection of entities = JPA infers @OneToMany
    @OneToMany(cascade = CascadeType.PERSIST)
    private List<OrderLine> orderLines = new ArrayList<>();

    // Required no-arg constructor
    public AnOrder() {}

    public AnOrder(Date creationDate) {
        this.creationDate = creationDate;
    }

    public void addOrderLine(OrderLine line) {
        this.orderLines.add(line);
    }

    // Getters and setters...
}
Why cascade = CascadeType.PERSIST? This tells JPA: "When I persist the Order, also persist its OrderLines automatically." Without this, you'd have to call em.persist() on each OrderLine separately.

Step 3 — Update persistence.xml

Register both entity classes. If you're using a separate project, create a new persistence.xml; if extending Project 1, add the new classes.

<persistence-unit name="Week2PU"
                  transaction-type="RESOURCE_LOCAL">
  <provider>org.eclipse.persistence.jpa.PersistenceProvider</provider>
  <class>week2.project2.AnOrder</class>
  <class>week2.project2.OrderLine</class>
  <properties>
    <property name="jakarta.persistence.jdbc.driver"
             value="com.mysql.cj.jdbc.Driver"/>
    <property name="jakarta.persistence.jdbc.url"
             value="jdbc:mysql://localhost:3306/week2db"/>
    <property name="jakarta.persistence.jdbc.user"
             value="root"/>
    <property name="jakarta.persistence.jdbc.password"
             value="your_password"/>

             value="drop-and-create"/>

</persistence-unit>
Both classes must be listed. If you forget to register OrderLine, JPA won't recognise it as an entity and the relationship mapping will fail.

Step 4 — Create Main.java for Project 2

Create an order, add two order lines, and persist everything in a single transaction.

package week2.project2;

import jakarta.persistence.*;
import java.util.Date;

public class Main {
    public static void main(String[] args) {
        EntityManagerFactory emf =
            Persistence.createEntityManagerFactory("Week2PU");
        EntityManager em = emf.createEntityManager();
        EntityTransaction tx = em.getTransaction();

        tx.begin();

        // Create order lines
        OrderLine line1 = new OrderLine("Jakarta EE Book", 49.99, 2);
        OrderLine line2 = new OrderLine("USB-C Cable", 12.50, 5);

        // Create order and add lines
        AnOrder order = new AnOrder(new Date());
        order.addOrderLine(line1);
        order.addOrderLine(line2);

        // Persist — cascade saves OrderLines too
        em.persist(order);
        System.out.println("Order persisted with ID: " + order.getId());

        tx.commit();
        em.close();
        emf.close();
    }
}

Troubleshooting — Project 2

OrderLine not persisted / only Order saved

You likely forgot cascade = CascadeType.PERSIST on the @OneToMany annotation. Without cascade, JPA only persists the Order and ignores its children.

"Unknown entity type: OrderLine"

Make sure <class>week2.project2.OrderLine</class> is listed in persistence.xml. Both entity classes must be registered.

No join table created

Check that the orderLines field type is List<OrderLine> (not just OrderLine). The Collection type triggers the join table strategy.

"Object: OrderLine is not a known entity type"

The @Entity annotation might be missing from OrderLine.java, or you imported the wrong @Entity (should be jakarta.persistence.Entity).

Quick Check

You have @OneToMany on a List<OrderLine> field but forgot cascade = CascadeType.PERSIST. You call em.persist(order). What happens?

A. Everything works fine — cascade is optional.
B. The Order is saved but OrderLines are not persisted — the join table will be empty.
C. A compile error — cascade is required on @OneToMany.
D. Both are saved but in separate transactions.
Correct! Without cascade, em.persist(order) only persists the Order entity. The OrderLine objects remain "new" (unmanaged) and are not written to the database. You'd either need cascade or explicit em.persist(line) calls for each OrderLine.
Not quite. Without cascade = CascadeType.PERSIST, only the Order itself is persisted. The OrderLine objects are ignored — they remain detached. You need either the cascade setting or separate em.persist() calls for each line.

Add a totalPrice field to AnOrder that computes the sum of all order line amounts. Mark it @Transient. Verify it doesn't appear in the database table.

Use @JoinTable(name = "order_items", joinColumns = ..., inverseJoinColumns = ...) to rename the join table and its FK columns. Verify the new names in MySQL.

Lab Summary

What You Learned

@Table overrides table names. @Column controls column names, lengths, nullability, and updatability. @OneToMany with a Collection field creates a join table by default. cascade = CascadeType.PERSIST auto-saves child entities.

Key Pattern

The workflow is always: Annotate → Persist → Verify in MySQL. JPA's configuration-by-exception means annotations are for overrides only — the defaults handle the common cases.
