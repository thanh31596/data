Moving from Propositional Logic to Predicate Logic

This week we extend our logical framework to handle statements about properties, relations, and quantities.

Why Do We Need More Than Propositional Logic?

Consider these arguments:

Argument 1:

‚Ä¢ If n is an even number, then n¬≤ is also even.

‚Ä¢ 6 is an even number.

‚Ä¢ Therefore, 6¬≤ is even.

Argument 2:

‚Ä¢ Everyone in the room is a suspect.

‚Ä¢ Poirot is in the room.

‚Ä¢ Therefore, Poirot is a suspect.

These are clearly valid arguments!

But we cannot express them properly using only propositional logic (p, q, ‚àß, ‚à®, ‚Üí, ¬¨, ‚Üî).

We need to talk about properties, variables, and quantification!

What's Missing from Propositional Logic?

Three key limitations:

1. Variables

We can't express "for any number x"

2. Properties

We can't express "x is even" as a general property

3. Quantities

We can't express "all", "some", "every", "there exists"

Predicate Logic extends propositional logic by adding:
  • Predicates (properties and relations)
  • Variables that range over domains
  • Quantifiers (‚àÄ for "all", ‚àÉ for "exists")

From Propositions to Predicates Foundation

Step 1: Start with specific propositions

‚Ä¢ "This apple is red" ‚Äì True or False

‚Ä¢ "The light is red" ‚Äì True or False

‚Ä¢ "My shoes are red" ‚Äì True or False

Step 4: Create a predicate

What is a Predicate? Foundation

Definition: Predicate

A predicate is a statement with one or more variables, such that:

  • Each variable can take values from a given universe (domain)
  • When we assign values to all variables, we get a proposition (something that is true or false)

Key Idea:

Predicate + Values = Proposition

Example: Let P(x) denote "x is greater than 5"

where x is a natural number.


‚Ä¢ P(x) by itself is NOT a proposition (it's neither true nor false)

‚Ä¢ P(7) is a proposition: ‚úì TRUE

‚Ä¢ P(3) is a proposition: ‚úó FALSE

Predicates with Multiple Variables Foundation

Predicates can express relationships

Example: One variable (property)

P(x) = "x is red"

‚Ä¢ P(this apple) ‚Äì True or False

‚Ä¢ This expresses a property of one thing

Example: Two variables (relation)

‚Ä¢ R(this apple, red) ‚Äì True or False

‚Ä¢ R(my shoes, blue) ‚Äì True or False

‚Ä¢ This expresses a relationship between two things

‚ö†Ô∏è Order Matters!

R(this apple, red) means "this apple has the colour red" ‚úì

R(red, this apple) means "red has the colour this apple" ‚úó (nonsense!)

The Importance of Domains/Universes Foundation

Definition: Universe (or Domain) of Definition

The universe or domain of a variable is the set of all possible values that the variable can take.

Example: S(x, y) means "x < y"

Case 1: x and y are natural numbers

‚Ä¢ S(3, 5) is TRUE

‚Ä¢ S(5, 3) is FALSE

‚Ä¢ S(-1, 1) is undefined (‚àí1 is not a natural number!)

‚Ä¢ S(3, 5) is TRUE

‚Ä¢ S(5, 3) is FALSE

‚Ä¢ S(-1, 1) is TRUE (now it makes sense!)

Always specify your universe clearly!

The Universal Quantifier ‚àÄ Foundation

Definition: Universal Quantification

Let P(x) be a predicate where x takes values in universe U.

‚àÄxP(x)

How to read ‚àÄxP(x):

  • "For all x, P(x)"
  • "For every x, P(x) is true"
  • "P(x) holds for all values of x"
  • "Every x satisfies P(x)"

Example:

Let P(n) = "n¬≤ ‚â• n" where n is an integer

‚àÄnP(n) means "Every integer n satisfies n¬≤ ‚â• n"

This is much cleaner than an infinite conjunction!

Universal Quantifier Examples Foundation

Example 1:

‚àÄxP(x) is TRUE

Every real number is less than itself plus one.

Example 2:

‚àÄxQ(x) is FALSE

Why? Because Q(0) is false (0 is not less than 0¬≤)

We only need one counterexample to disprove ‚àÄxP(x)!

üéØ Key Pattern: Proving/Disproving ‚àÄxP(x)

To PROVE ‚àÄxP(x) is TRUE: Must show P(x) is true for EVERY x in the universe

To DISPROVE ‚àÄxP(x): Only need to find ONE x where P(x) is false

The Existential Quantifier ‚àÉ Foundation

Let P(x) be a predicate where x takes values in universe U.

‚àÉxP(x)

  • "At least one x satisfies P(x)"

Example:

where x is a student in this class

‚àÉxP(x) means "At least one student has their phone not on silent"

Existential Quantifier Examples Foundation

Example 1:

P(x) = "x < x + 1" where x is a real number

‚àÉxP(x) is TRUE

Example 2:

‚àÉxQ(x) is TRUE

üéØ Key Pattern: Proving/Disproving ‚àÉxP(x)

Relationship Between ‚àÄ and ‚àÉ

Important Logical Relationship

If P(x) is true for every x in a non-empty universe U, then P(x) is true for at least one x in U.

‚àÄxP(x) ‚Üí ‚àÉxP(x)

(This implication is true)

‚ö†Ô∏è But the converse is NOT generally true!

‚àÉxP(x) does NOT imply ‚àÄxP(x)


Example:

‚Ä¢ ‚àÉxP(x) is TRUE (because 2 is even)

‚Ä¢ ‚àÄxP(x) is FALSE (because 3 is not even)

‚àÄxP(x)

ALL elements satisfy P

‚Üí

‚àÉxP(x)

AT LEAST ONE satisfies P

Edge Case: The Empty Universe

When the universe has NO elements:

‚àÄxP(x) is always TRUE (vacuously true)

‚àÉxP(x) is always FALSE

Example: Unicorns

Let U = {all unicorns in reality}

Let P(x) = "x has golden wings"


‚àÄxP(x): "All unicorns have golden wings" is TRUE!

Why? There are no unicorns to check, so the statement is vacuously true.


‚àÉxP(x): "There exists a unicorn with golden wings" is FALSE

Why? There are no unicorns, so we can't find one with this property.

This is why mathematicians say "All unicorns are pink" is technically a true statement!

Scope and Precedence Building

Scope of a Quantifier

The scope of a quantifier is the part of the expression to which the quantifier applies.

‚àÄx(F(x) ‚àß G(x))

The scope of ‚àÄx is the entire expression (F(x) ‚àß G(x))

Precedence Rules (High to Low):

  1. Parentheses ()
  2. Quantifiers: ‚àÄ, ‚àÉ
  3. Negation: ¬¨
  4. Conjunction: ‚àß
  5. Disjunction: ‚à®
  6. Implication: ‚Üí
  7. Biconditional: ‚Üî

‚ö†Ô∏è Be Careful!

‚àÄxF(x) ‚àß G(x) means (‚àÄxF(x)) ‚àß G(x)

The quantifier only applies to F(x), not to G(x)!

This is a predicate, not a proposition (x in G(x) is free)

Bound vs Free Variables

Definitions

Bound variable: A variable that is controlled by a quantifier

Free variable: A variable that is NOT controlled by a quantifier

Example 1: All variables bound

‚àÄx(F(x) ‚àß G(x))

‚Ä¢ The variable x appears in F(x) and G(x)

‚Ä¢ x is bound by ‚àÄx

‚Ä¢ This is a proposition (no free variables)

Example 2: Mixed bound and free

‚àÄxF(x) ‚àß G(x)

‚Ä¢ The first x (in F(x)) is bound by ‚àÄx

‚Ä¢ The second x (in G(x)) is free (not in scope of ‚àÄx)

‚Ä¢ This is a predicate (has a free variable)

üéØ Key Rule

A statement is a proposition only if ALL variables are bound.

If there are any free variables, it's a predicate.

Expressing Uniqueness Challenge

Shorthand Notation

"There exists an x such that P(x), and for all x and y, if both satisfy P, then they're equal"
