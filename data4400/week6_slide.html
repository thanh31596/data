Recap: Facebook Prophet

Prophet = Trend + Seasonality + Holidays + Error

Key Features:

  • Automatic seasonality detection
  • Handles missing data gracefully
  • Built-in holiday effects
  • Changepoint detection
  • Uncertainty intervals

When to Use Prophet:

  • Multiple seasonality patterns
  • Strong holiday effects
  • Historical data with gaps
  • Need for interpretable parameters
  • Business forecasting at scale
5 / 31

Recap: Time Series Decomposition

Additive Model

Y(t) = Trend + Seasonal + Error

Use when seasonal fluctuations are constant

Multiplicative Model

Y(t) = Trend × Seasonal × Error

Use when seasonal fluctuations change with level

Key Difference

Additive: Constant seasonal amplitude

Multiplicative: Proportional seasonal amplitude

Choose based on your data pattern!

Recap: Moving Average Smoothing

MA(k)ₜ = (yₜ + yₜ₋₁ + ... + yₜ₋ₖ₊₁) / k

Advantages:

  • Simple to implement and understand
  • Effective noise reduction
  • No parameters to optimize

Limitations:

  • Equal weights to all observations
  • Lag in detecting trend changes
  • Cannot forecast beyond data

Exponential Smoothing Fundamentals

Weighting Principle

Exponentially decreasing weights: newest data gets highest weight

Short-term Focus

Optimized for short-term predictions (1-3 periods ahead)

Univariate Approach

Works with single time series variable (no external factors)

Simple Exponential Smoothing (SES)

Ŷₜ₊₁ = αYₜ + (1-α)Ŷₜ

When to Use:

  • No clear trend in data
  • No seasonal patterns
  • Level changes slowly over time
  • Short-term forecasting needed

Alpha (α) Parameter:

α → 1: More reactive (recent data dominates)

α → 0: More stable (historical average)

Double Exponential Smoothing (Holt's Method)

Level: Lₜ = αYₜ + (1-α)(Lₜ₋₁ + Tₜ₋₁)
Trend: Tₜ = β(Lₜ - Lₜ₋₁) + (1-β)Tₜ₋₁
Forecast: Ŷₜ₊ₕ = Lₜ + hTₜ

Purpose:

Handles data with trends but no seasonality

Two Parameters:

  • α: Level smoothing (0 < α < 1)
  • β: Trend smoothing (0 < β < 1)

Applications:

  • Sales with growth trends
  • Economic indicators
  • Technology adoption curves
13 / 32

Triple Exponential Smoothing (Holt-Winters)

Level: Cₜ = α(Yₜ - Sₜ₋ₚ) + (1-α)(Cₜ₋₁ + Tₜ₋₁)
Trend: Tₜ = β(Cₜ - Cₜ₋₁) + (1-β)Tₜ₋₁
Seasonal: Sₜ = γ(Yₜ - Cₜ) + (1-γ)Sₜ₋ₚ
Forecast: Ŷₜ₊ₖ = Cₜ + kTₜ + Sₜ₊ₖ₋ₚ

Purpose: Complete seasonal forecasting solution

α: Level parameter | β: Trend parameter | γ: Seasonal parameter

Model Evaluation Methods

Train/Test Split: 80% / 20% Rule

Essential for detecting overfitting and validating forecast accuracy

RMSE

√(Σ(yᵢ - ŷᵢ)² / n)

Penalizes large errors more heavily

MAE

Σ|yᵢ - ŷᵢ| / n

Average absolute error

MAPE

100 × Σ|yᵢ - ŷᵢ|/yᵢ / n

Percentage error (scale-free)

Remember: Lower values = Better forecasts!

Implementing Holt-Winters in Tableau

Step-by-Step Process:

Common Pitfalls in Exponential Smoothing

Overfitting

Using complex models on simple data

Solution: Start simple, add complexity only if needed

Wrong Model Choice

Using SES for trending data

Solution: Analyze data patterns first

Parameter Misuse

Setting α = 1 for noisy data

Solution: Use optimization or cross-validation

Remember: Simpler models often outperform complex ones for short-term forecasts!

23 / 32

Choosing Smoothing Parameters

Manual Selection Guidelines:

  • Stable data: α = 0.1-0.3
  • Trending data: β = 0.1-0.2
  • Strong seasonality: γ = 0.5-0.8
  • Noisy data: Lower all parameters

Automated Optimization:

  • Grid search over parameter space
  • Minimize forecast error (RMSE)
  • Use cross-validation
  • Most software does this automatically

Best Practices for Implementation

Do's:

  • Start with simple models
  • Use sufficient historical data (2+ seasons)
  • Validate on hold-out sample
  • Update parameters periodically
  • Consider multiple models
  • Document assumptions

Don'ts:

  • Overfit to historical data
  • Ignore data quality issues
  • Use for long-term forecasts
  • Set parameters arbitrarily
  • Ignore structural breaks
  • Forget to monitor performance
