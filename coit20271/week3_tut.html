Scene Setup in Unity

Start with a fresh 3D (URP) project. We will build the entire scene manually so every GameObject's purpose is clear.

Create the Ground Plane

1

Go to GameObject → 3D Object → Plane. Name it Ground.

2

In Inspector, set Scale → (5, 1, 5) — this gives a 50×50 unit arena.

3

Create a new Material: Right-click in Project → Create → Material. Set Albedo to a grey colour (#888888). Drag onto Ground.

Create the Player

1

GameObject → 3D Object → Sphere. Name it Player. Set Position to (0, 0.5, 0).

2

Add Component: Rigidbody. In Rigidbody settings, set Constraints → Freeze Rotation X, Y, Z (prevents tipping over). Also set Drag = 5 (gives snappy stop).

3

Create a red material (#CC0000) and apply it to Player.

Camera Rig Structure

The CameraRig is an empty parent. The Camera sits 3 units up and 6 units behind. In LateUpdate, the rig position lerps toward the Player.

1

GameObject → Create Empty. Name it CameraRig. Set Position to (0, 0, 0).

2

In the Hierarchy, drag Main Camera onto CameraRig — it becomes a child.

3

With Main Camera selected, set local Position to (0, 3, -6) and Rotation to (20, 0, 0). This aims it slightly down at the player.

Why a CameraRig? By rotating the rig (not the camera), the camera orbits around the player cleanly. The offset stays consistent at all angles.

PlayerController.cs — Movement & Shooting

Create Assets/Scripts/PlayerController.cs and attach it to the Player GameObject. This script handles WASD movement via Rigidbody forces, Space to shoot, and a fire-rate cooldown.

Input → Direction Vector → AddForce → Rigidbody moves player

PlayerController.cs Assets/Scripts/PlayerController.cs
using UnityEngine;

public class PlayerController : MonoBehaviour
{
    // ── Inspector-exposed fields ───────────────────────────
    [Header("Movement")]
    public float moveSpeed = 15f;       // Force applied per frame

    [Header("Shooting")]
    public GameObject bulletPrefab;      // Drag Bullet prefab here
    public Transform  firePoint;         // Empty child at front of player
    public float bulletSpeed  = 20f;
    public float fireRate     = 0.3f;    // Seconds between shots

    // ── Private state ─────────────────────────────────────
    private Rigidbody rb;
    private float nextFireTime = 0f;

    void Start()
    {
        rb = GetComponent<Rigidbody>();
    }

    // FixedUpdate is used for physics — runs at fixed time steps
    void FixedUpdate()
    {
        HandleMovement();
    }

    // Update is used for input polling — runs every rendered frame
    void Update()
    {
        if (Input.GetKey(KeyCode.Space))
            TryShoot();
    }

    void HandleMovement()
    {
        // Read axes: returns -1 to +1 smoothly
        float h = Input.GetAxis("Horizontal");  // A/D or Left/Right
        float v = Input.GetAxis("Vertical");    // W/S or Up/Down

        // Build a flat direction vector (Y=0 keeps player on ground)
        Vector3 direction = new Vector3(h, 0f, v).normalized;

        // Apply force relative to world space
        rb.AddForce(direction * moveSpeed, ForceMode.Force);
    }

    void TryShoot()
    {
        // Cooldown check: only fire if enough time has passed
        if (Time.time < nextFireTime) return;
        nextFireTime = Time.time + fireRate;

        // Spawn bullet at firePoint position and rotation
        GameObject bullet = Instantiate(
            bulletPrefab,
            firePoint.position,
            firePoint.rotation
        );

        // Launch bullet forward (firePoint faces the shooting direction)
        Rigidbody bRb = bullet.GetComponent<Rigidbody>();
        bRb.velocity = firePoint.forward * bulletSpeed;

        // Safety: destroy bullet after 3 seconds to avoid memory leak
        Destroy(bullet, 3f);
    }
}
Common mistake: Using Update() for physics forces (AddForce) causes jittery movement. Always use FixedUpdate() for Rigidbody operations.

CameraFollow.cs — Third-Person Camera

Create Assets/Scripts/CameraFollow.cs and attach it to CameraRig (the empty parent — not the camera itself). This script uses Vector3.Lerp for smooth lag and runs in LateUpdate() so it executes after all physics.

Lerp moves the camera a percentage of the remaining distance each frame — creating natural deceleration/lag

CameraFollow.cs Assets/Scripts/CameraFollow.cs
using UnityEngine;

public class CameraFollow : MonoBehaviour
{
    [Header("Target")]
    public Transform target;              // Drag Player here
    public float smoothSpeed = 8f;        // Higher = snappier follow

    // LateUpdate runs AFTER all Update() and FixedUpdate() calls
    // This prevents camera jitter caused by physics lag
    void LateUpdate()
    {
        if (target == null) return;

        // Smoothly move the RIG to match the player's XZ position
        // We only follow X and Z — keeping Y fixed prevents bobbing
        Vector3 targetPos = new Vector3(
            target.position.x,
            transform.position.y,  // Keep rig at its own Y height
            target.position.z
        );

        transform.position = Vector3.Lerp(
            transform.position,    // Current rig position
            targetPos,             // Where we want to be
            smoothSpeed * Time.deltaTime  // How fast to catch up
        );
    }
}

Why LateUpdate?

Player physics run in FixedUpdate. Camera must sample the final position after all movement is resolved — otherwise you get one-frame jitter.

Why only follow X/Z?

We don't want the rig to bob vertically as the sphere rolls. Lock Y on the rig so only the camera's local offset provides the height.

Camera Setup in Inspector

Set FOV = 60, Near Clip = 0.3, Far Clip = 300. This covers the full 50×50 arena without performance issues.

Bullet.cs — Projectile & Hit Detection

Create Assets/Scripts/Bullet.cs. Create a Sphere primitive (scale 0.2), add a Rigidbody (no gravity), attach this script, then save it as a Prefab in Assets/Prefabs.
Bullet.cs Assets/Scripts/Bullet.cs
using UnityEngine;

public class Bullet : MonoBehaviour
{
    // Called when this collider touches another collider
    void OnTriggerEnter(Collider other)
    {
        // Check if we hit something tagged "Target"
        if (other.gameObject.CompareTag("Target"))
        {
            // Tell GameManager to add a point
            GameManager.instance.AddScore(10);

            // Destroy the target it hit
            Destroy(other.gameObject);

            // Spawn a replacement target after 1 second
            GameManager.instance.SpawnTarget(1f);
        }

        // Destroy the bullet regardless of what was hit
        // (except the Player itself — we check the tag)
        if (!other.gameObject.CompareTag("Player"))
            Destroy(gameObject);
    }
}

// ─────────────────────────────────────────────────────
//  BULLET SETUP CHECKLIST (do this on the Bullet prefab)
//  1. Add Rigidbody → Use Gravity = OFF
//  2. Set Collider → Is Trigger = ON  (lets OnTriggerEnter fire)
//  3. Attach this Bullet.cs script
//  4. Drag into Assets/Prefabs to create the prefab
// ─────────────────────────────────────────────────────

OnTriggerEnter fires when the bullet's trigger collider overlaps the target — then all three consequences happen in sequence

Collider SettingIsTriggerEffect
Bullet ColliderONPasses through geometry, fires OnTriggerEnter
Target ColliderOFFSolid box — triggers bullet's OnTriggerEnter when overlapped
Ground ColliderOFFSolid plane — player and targets rest on it
Player ColliderOFFSolid sphere — Rigidbody rests on ground

TargetMover.cs — Moving Targets

Targets patrol between two points using Mathf.PingPong — a built-in Unity function that bounces a value between 0 and a max, making targets reverse direction automatically without any state machine.

PingPong returns a value that bounces between 0 and the range — we use this to Lerp position between two waypoints

TargetMover.cs Assets/Scripts/TargetMover.cs
using UnityEngine;

public class TargetMover : MonoBehaviour
{
    [Header("Patrol Points")]
    public Vector3 pointA = new Vector3(-8f, 0.5f, 0f);
    public Vector3 pointB = new Vector3( 8f, 0.5f, 0f);
    public float speed = 2.5f;

    void Update()
    {
        // PingPong bounces a value between 0 and 1 over time
        float t = Mathf.PingPong(Time.time * speed, 1f);

        // Lerp moves the target smoothly between the two points
        transform.position = Vector3.Lerp(pointA, pointB, t);
    }

    // Draw the patrol path in the Scene view (editor only)
    void OnDrawGizmos()
    {
        Gizmos.color = Color.red;
        Gizmos.DrawSphere(pointA, 0.2f);
        Gizmos.DrawSphere(pointB, 0.2f);
        Gizmos.DrawLine(pointA, pointB);
    }
}
Making Targets Prefabs: Create a Cube (scale 1×1×1), apply a red material, add a Box Collider, set Tag to "Target", attach TargetMover.cs, then drag to Assets/Prefabs. Spawn via Instantiate(targetPrefab, spawnPos, Quaternion.identity).

Lighting Setup for the Game Scene

Apply this week's lighting knowledge. The scene uses a Directional Light as the sun, a Point Light above the player for dramatic fill, and tuned Ambient settings.

Two-light setup: Directional (sun angle) provides main shadows; Point Light provides overhead fill to avoid pitch-black shadows. *Make Point Light a child of Player so it follows.

Step-by-Step Lighting Setup

1

Select the default Directional Light. Set Rotation to (50, -30, 0). Set Shadow Type → Soft Shadows. Set Intensity to 1.2.

2

GameObject → Light → Point Light. Set Position to (0, 4, 0). Set Range = 12, Intensity = 0.6, colour to a soft white/blue (#DDF).

3

Open Window → Rendering → Lighting. Under Environment, set Ambient Color to a dark grey (#333). This prevents pure-black shadows on mobile.

4

Optional: Drag the Point Light onto the Player GameObject to make it a child. It will then follow the player around the arena.

Week 3 connection: This directly applies today's lecture — you're using a Directional Light for sun simulation, a Point Light for fill, adjusting Shadow Type between Hard/Soft, and setting Ambient. These are all Inspector properties from Slide 3.

GameManager.cs — Score, Lives & UI

Create Assets/Scripts/GameManager.cs and attach it to an empty GameObject called GameManager. It uses a Singleton pattern so any script can call GameManager.instance.AddScore() from anywhere.
GameManager.cs Assets/Scripts/GameManager.cs
using UnityEngine;
using UnityEngine.UI;
using System.Collections;

public class GameManager : MonoBehaviour
{
    // ── Singleton pattern ──────────────────────────────────
    // Lets any script call GameManager.instance.AddScore()
    public static GameManager instance;

    [Header("Game State")]
    public int score = 0;
    public int lives = 3;

    [Header("UI References")]
    public Text scoreText;    // Drag UI Text here
    public Text livesText;    // Drag UI Text here

    [Header("Spawning")]
    public GameObject targetPrefab;   // Drag Target prefab here
    public Transform[] spawnPoints;   // Array of spawn locations

    void Awake()
    {
        // Singleton: ensure only one GameManager exists
        if (instance == null)
            instance = this;
        else
            Destroy(gameObject);
    }

    void Start()
    {
        // Spawn initial targets at game start
        for (int i = 0; i < spawnPoints.Length; i++)
            SpawnTargetAt(i);

        UpdateUI();
    }

    public void AddScore(int points)
    {
        score += points;
        UpdateUI();
    }

    public void LoseLife()
    {
        lives--;
        UpdateUI();
        if (lives <= 0) GameOver();
    }

    // Spawn a target after a delay (called from Bullet.cs)
    public void SpawnTarget(float delay = 0f)
    {
        StartCoroutine(SpawnAfterDelay(delay));
    }

    IEnumerator SpawnAfterDelay(float delay)
    {
        yield return new WaitForSeconds(delay);
        int idx = Random.Range(0, spawnPoints.Length);
        SpawnTargetAt(idx);
    }

    void SpawnTargetAt(int index)
    {
        Instantiate(targetPrefab, spawnPoints[index].position, Quaternion.identity);
    }

    void UpdateUI()
    {
        if (scoreText) scoreText.text = "Score: " + score;
        if (livesText) livesText.text = "Lives: " + lives;
    }

    void GameOver()
    {
        Debug.Log("Game Over! Final score: " + score);
        // In a full project: load a GameOver scene or show a panel
        Time.timeScale = 0f;  // Pause the game
    }
}

Extension: Add a SpotLight

Make the bullet a light source! Attach a small SpotLight to the Bullet prefab (intensity 2, range 4, color yellow). It briefly illuminates whatever it's flying toward.

Extension: Camera FOV on Shoot

In PlayerController, add a brief FOV kick: camera.fieldOfView = Mathf.Lerp(fov, fov-5, 0.5f) when shooting — gives a firing recoil feel.

Extension: Bake Lightmap

Mark Ground as Static. Open Window → Rendering → Lighting → Generate Lighting. Baking removes the runtime shadow cost from mobile builds entirely.

Extension: Cinemachine

Replace CameraFollow.cs with a Cinemachine Virtual Camera. Set Follow = Player, Look At = Player, Body = Transposer offset (0,3,−6). Get free smoothing and collision handling.
